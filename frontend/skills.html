云计算部署思路

本个人主页最初只是部署在阿里云 ECS 上的单机 Web 应用，前端页面、FastAPI 后端和 SQLite 数据库都运行在同一台服务器中。 为了进一步体现云计算课程中的核心思想，我对系统进行了云计算化改造，使它从普通个人主页升级为 “Docker 多实例 + Nginx 负载均衡 + Redis 缓存”的轻量级云计算实践平台。

当前服务器配置为 2 核 4G，系统通过 Docker Compose 启动两个 FastAPI 后端实例 api1 和 api2， 分别监听 8001 与 8002 端口。Nginx 使用 upstream 后端集群，将所有 /api/ 请求轮询转发到两个后端实例， 通过 /api/instance 接口可以观察到请求会命中不同的后端容器。

在性能优化方面，系统引入 Redis 缓存层，对访问频率较高的 /api/messages 留言接口进行缓存。 首次访问时，后端从 SQLite 数据库读取留言列表并写入 Redis；后续访问优先从 Redis 获取缓存数据， 从而减少数据库读取压力。当新增留言或回复留言后，系统主动删除缓存，使下一次请求重新读取数据库， 兼顾访问性能和数据最终一致性。

Docker 容器化

将 FastAPI 后端封装为 Docker 镜像，提高部署一致性，减少环境差异带来的问题。

Nginx 负载均衡

通过 upstream 将请求分发到 api1 和 api2，实现多实例访问与基础流量分摊。

基础容错设计

当其中一个后端容器停止时，另一个实例仍可继续提供接口服务，提升系统可用性。

Redis 缓存优化

缓存留言列表接口，减少 SQLite 读取压力，并通过主动删除缓存保证数据更新后的最终一致性。

可检查接口： /api/instance /api/cloud/status /api/cache/status /api/messages
